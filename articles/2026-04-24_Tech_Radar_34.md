# Tech Radar Vol. 34

Summary of selected blips from the [Technology Radar Vol. 34](https://www.thoughtworks.com/radar/), organized by quadrant and ring. This volume is dominated by the industrialization of AI coding agents — context engineering, agent skills, feedback loops — and the corresponding risks (instruction bloat, cognitive debt, misleading productivity metrics).

## Techniques

### Adopt

* **[Context engineering][ctx-eng]**:
  * Context engineering has shifted from tactical prompt optimization to a foundational architectural concern for AI systems. Teams move from static prompts to dynamic, progressive context disclosure, loading only necessary information to reduce hallucinations and improve agent reasoning.
  * 💡 Very important for harness engineering.
* **[Curated shared instructions for software teams][curated-instr]**:
  * Treat AI guidance as a collaborative engineering asset by anchoring curated instructions (e.g. `CLAUDE.md`, `AGENTS.md`) directly into service templates, so every project starts with consistent, high-quality AI assistance rather than ad-hoc prompts.
  * 💡 Very important for harness engineering.

### Trial

* **[Agent Skills][agent-skills]**:
  * An open standard for packaging instructions, scripts and resources that are loaded only when needed, addressing context window exhaustion in AI agents.
  * 💡 Very important for harness engineering.
* **[Feedback sensors for coding agents][feedback-sensors]**:
  * Deterministic checks (compilers, linters, tests, custom validators) that provide real-time quality feedback during agent sessions, enabling self-correction before code is committed and reducing the need for human steering.
  * 💡 Very important for harness engineering.
* **[Progressive context disclosure][progressive-disclosure]**:
  * Agents go through a lightweight discovery phase to identify relevant information before loading detailed instructions into the context window, preventing context bloat and keeping signal-to-noise ratios sharp.
  * 💡 Very important for harness engineering.
* **[Mutation testing][mutation-testing]**:
  * Injects deliberate bugs into source code to verify that tests actually catch behavioral changes — a more honest fault-detection metric than code coverage, and particularly valuable when evaluating AI-generated test suites.

### Assess

* **[Architecture drift reduction with LLMs][arch-drift]**:
  * Combines deterministic analysis with LLM-powered evaluation to detect structural and semantic violations, then uses LLMs to help fix them — a kind of "garbage collection" for the entropy that AI-accelerated development can introduce.
* **[HTML tools][html-tools]**:
  * Packaging shareable utilities as single HTML files that run directly in the browser, avoiding the overhead of distributing CLIs or building full web apps — a portable format for lightweight task-specific tools produced by agentic workflows.
  * 💡 Interesting new technique, already used at Praxedo (PSS Utilities).
  * Relates to Moldable Development and the idea of "throwaway" tools that solve a specific problem without long-term maintenance commitment.
* **[Feedback flywheel][feedback-flywheel]**:
  * A continuous improvement cycle where successes and failures from agent sessions are captured and fed back into refined instructions and stronger feedback mechanisms for future runs.
  * 💡 Very important for harness engineering.
* **[Skills as executable onboarding documentation][skills-onboarding]**:
  * Uses Agent Skills to package instructions, scripts and docs as executable onboarding materials that adapt to the current state of the codebase — documentation becomes self-validating, making it obvious when guidance goes stale.
  * 💡 Interesting technique that could be applied at Praxedo.

### Caution

* **[Agent instruction bloat][instr-bloat]**:
  * Context files like `AGENTS.md` accumulate instructions over time, producing conflicting guidance that gets lost in long contexts. Be deliberate and selective, and lean on progressive context disclosure to surface only what the agent needs.
  * 💡 Very important for harness engineering.
* **[Codebase cognitive debt][cognitive-debt]**:
  * The growing gap between a system's implementation and the team's shared understanding of it. AI-accelerated change velocity can outstrip human comprehension of design intent, making systems harder to reason about and more failure-prone.
  * 💡 Very important for harness engineering.
* **[Coding throughput as a measure of productivity][throughput]**:
  * Lines of code or PR volume are misleading productivity metrics in AI-assisted development. Prefer first-pass acceptance rates and DORA metrics to capture real delivery impact.

## Platforms

### Trial

* **[Port][port]**:
  * Commercial internal developer portal that centralizes software assets, automates workflows and enforces engineering standards, exposing self-service capabilities (templates, APIs, automations, agents) through IDE and portal interfaces.

### Assess

* **[MCP Apps][mcp-apps]**:
  * First official MCP extension letting servers return interactive HTML interfaces (dashboards, forms, visualizations) directly inside conversations, via a standardized `ui://` resource scheme, with graceful degradation for hosts without UI support.

## Tools

### Adopt

* **[Claude Code][claude-code]**:
  * Anthropic's agentic coding tool, now in Adopt. Teams use it in production for complex multi-step workflows that extend beyond code into specs, infrastructure and documentation — it serves as a capability benchmark for coding agents.
  * 💡 Claude Code passe en Adopt.
* **[mise][mise]**:
  * Rust-based tool consolidating tool versioning, environment variables and task execution into a single declarative config. Recommended as the default for standardizing developer environments, especially polyglot setups.
  * 💡 Avec la fin de vie des IDE, il va falloir ré-industrialiser les environnements de dev.

### Trial

* **[Dev Containers][dev-containers]**:
  * Standardized `devcontainer.json`-based reproducible development environments, increasingly valued as sandboxed execution environments for coding agents — isolating AI tools from the host while preserving security boundaries.

### Assess

* **[Entire CLI][entire-cli]**:
  * Hooks into Git workflows to capture AI agent sessions (transcripts, prompts, tool calls, token usage) as searchable metadata on a dedicated repo branch. Its checkpoint system allows teams to rewind to known-good states when an agent deviates.
* **[Ty][ty]**:
  * Fast Rust-based Python type checker and language server from the Astral ecosystem, alongside [uv][uv] and [ruff][ruff] (both in Adopt). Provides rapid feedback to catch mistakes early — especially valuable as agentic coding grows.

### Caution

* **[OpenClaw][openclaw]**:
  * Open-source hyper-personal AI assistant that executes tasks via connected tools with persistent memory. Flagged for significant security risk due to the breadth of permissions it needs to function.

## Languages and Frameworks

### Adopt

* **[Apache Iceberg][iceberg]**:
  * Open table format for large-scale analytical data — defining how data files, metadata and schemas are organized on object storage (S3, etc.). Now supported across major cloud providers, with snapshot isolation and concurrent-write safety making it a solid default for modern data platforms.

### Assess

* **[Superpowers][superpowers]**:
  * A workflow built from composable skills that wraps coding agents in structured processes (planning, TDD, debugging, review). Distributed as a plugin for Claude Code and Cursor marketplaces.
  * 💡 To investigate.
